# Configuration Service Protocol Specification

This document describes the protocol used to configure devices in the vehicle-tracker system via BLE. It defines the GATT service structure, message format, and expected device behavior.

> **Note**: This document is still under development. The protocol may change as the system evolves.

## Overview

The Configuration Service allows authorized clients to read and write device configuration parameters via BLE. All messages are exchanged in JSON format, and the device expects messages to be well-formed and adhere to the defined schema.

For a full list of configuration parameters, refer to the [Device Configuration Overview](overview.md).

## Message Format

All characteristic values are JSON encoded.

### Write Request

To update a parameter, write a JSON message with a `value` field to the corresponding characteristic.

Example of a write message to update the MQTT Broker URL:
```json
{
  "value": "mqtt://broker.example.com"
}
```

### Responses

The device will always respond with a JSON message indicating the success or failure of the operation.

Example of a successful response:
```json
{
  "status": "success",
  "message": "MQTT Broker URL updated successfully."
}
```

Example of an error response:
```json
{
  "status": "error",
  "message": "Invalid MQTT Broker URL format."
}
```

## Authentication Operations

The device supports operations for generating a Certificate Signing Request (CSR) and installing a signed certificate for authentication. These operations are performed via the `device_csr` and `device_certificate` characteristics, respectively.

### Reading the CSR

Read the `device_csr` characteristic to obtain the PEM-encoded CSR generated by the device. This CSR can then be submitted to the Identity Service to obtain a signed certificate.

Example response:
```json
{
  "csr": "-----BEGIN CERTIFICATE REQUEST-----\n..."
}
```

### Installing the Certificate

Write the signed certificate to the `device_certificate` characteristic to install it on the device.

Example write message:
```json
{
  "value": "-----BEGIN CERTIFICATE-----\n..."
}
```
